# 因果分析 · Causal Analysis

> **作者：依观/yiguan**

## 这是什么

一个用于AI Agent的分析框架——给定任何结果、现象或产物，自动倒推完整的因果链条，找到改变结果的最短路径。

## 核心理念

> 每一个结果都是一系列因和缘组成。我们往往只能看见果，却看不见因缘。**因缘才是这条链条最有价值的地方。**

- **因**：必然条件——没有它，结果不会发生
- **缘**：偶然条件——有了它，结果才以这种特定方式呈现
- 一颗种子是因，阳光雨水是缘，开花是果。同样的因，不同的缘，果就不同

### 最强用法：找漏洞，不造新链

> 找原来"因"的漏洞，可比重新创造一条新的因果链条快捷得多了。

因果分析的真正威力不是"理解现状"，而是**找到现有因果链中最脆弱的因**——那个一旦去掉或替换，整条链就会崩塌的节点。找到它，你就有了改变结果的最短路径。

- 重新造一条链 = 从零开始，成本巨大，不确定性高
- 找到旧链的漏洞 = 改一个节点，果就变了，四两拨千斤

## 分析维度

1. **因** — "没有它，结果不会发生"（用反事实测试判定）
2. **缘** — "有了它，结果才长成这样"（用替换测试判定）
3. **利益链条** — 谁需要它、谁制作它、谁受益、谁受损
4. **构成要素** — 它由什么组成、怎么来的
5. **利弊权衡** — 这样做获得了什么、牺牲了什么（含隐性代价/暗债）
6. **历史因果** — 为什么是现在、为什么是这样
7. **最脆弱的因** — 杠杆率最高的节点，改变成本极低但会导致整条链崩塌
8. **假设失效** — 如果推演错了，最可能错在哪

## 怎么用

### 作为AI Agent技能

将 `SKILL.md` 放入你的Agent技能目录（如 Hermes Agent 的 `~/.hermes/skills/`），Agent加载后即可对任何输入执行完整的因果分析。

### 作为分析框架

按SKILL.md中的分析框架和执行规则，手动对任何问题进行逐层拆解。交付前必查清单确保分析质量。

## 关键规则

- **先搜索再分析** — 不凭记忆断言，用搜索验证事实
- **5Why追问** — 连续问5次"为什么"，直到触及物理定律或人性底层
- **区分因和缘** — 因=去掉则果必不发生，缘=替换则果的形态变但本质不变
- **防范因果倒置和相关当因果** — 必须找到作用机制
- **找隐性代价（暗债）** — 任何方案都有被悄悄牺牲的东西
- **最脆弱的因必须从因果链推导** — 不能凭空编造

## 适用场景

不限于任何领域——产品分析、技术选型、商业现象、制度规则、自然现象，只要有"果"可寻，就能倒推"因缘"。
